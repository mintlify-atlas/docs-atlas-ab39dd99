## List Users

<api-endpoint method="GET" path="/admin/users" />

Get a paginated list of all users with optional search functionality.

### Query Parameters

<ParamField query="limit" type="integer" default="20">
  Number of users to return per page (1-100)
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of users to skip for pagination
</ParamField>

<ParamField query="search" type="string" optional>
  Search query to filter users by email or other fields
</ParamField>

### Response

<ResponseField name="users" type="array">
  Array of user objects
  <expandable>
    <ResponseField name="id" type="uuid">
      Unique user identifier
    </ResponseField>
    <ResponseField name="email" type="string">
      User email address
    </ResponseField>
    <ResponseField name="is_active" type="boolean">
      Whether the user account is active
    </ResponseField>
    <ResponseField name="role_id" type="uuid" optional>
      Legacy single role ID (may be null)
    </ResponseField>
    <ResponseField name="role_name" type="string" optional>
      Legacy role name (may be null)
    </ResponseField>
    <ResponseField name="roles" type="array">
      Array of assigned roles
      <expandable>
        <ResponseField name="role_id" type="uuid">
          Role identifier
        </ResponseField>
        <ResponseField name="role_name" type="string">
          Role name
        </ResponseField>
        <ResponseField name="assigned_at" type="string" optional>
          ISO 8601 timestamp when role was assigned
        </ResponseField>
      </expandable>
    </ResponseField>
  </expandable>
</ResponseField>

<ResponseField name="total" type="integer">
  Total number of users matching the query
</ResponseField>

<ResponseField name="limit" type="integer">
  Number of users returned per page
</ResponseField>

<ResponseField name="offset" type="integer">
  Number of users skipped
</ResponseField>

<ResponseField name="has_more" type="boolean">
  Whether there are more users to fetch
</ResponseField>

## Get User Effective Permissions

<api-endpoint method="GET" path="/admin/users/{user_id}/permissions/effective" />

Get user's effective permissions with breakdown by source (roles, direct sets, direct permissions).

### Path Parameters

<ParamField path="user_id" type="uuid" required>
  The user ID to query
</ParamField>

### Response

<ResponseField name="user_id" type="uuid">
  The user identifier
</ResponseField>

<ResponseField name="from_roles" type="array">
  Permissions inherited from roles
</ResponseField>

<ResponseField name="from_direct_sets" type="array">
  Permissions from directly assigned permission sets
</ResponseField>

<ResponseField name="from_direct_permissions" type="array">
  Directly assigned permissions
</ResponseField>

<ResponseField name="effective" type="array">
  Combined effective permissions (union of all sources)
</ResponseField>

## Get User RBAC History

<api-endpoint method="GET" path="/admin/users/{user_id}/rbac-history" />

Get all RBAC changes related to a specific user.

### Path Parameters

<ParamField path="user_id" type="uuid" required>
  The user ID to query
</ParamField>

### Query Parameters

<ParamField query="limit" type="integer" default="50">
  Number of history entries to return (1-100)
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of entries to skip for pagination
</ParamField>

### Response

<ResponseField name="status" type="string">
  Response status
</ResponseField>

<ResponseField name="user_id" type="string">
  The user ID as a string
</ResponseField>

<ResponseField name="history" type="array">
  Array of RBAC change events
  <expandable>
    <ResponseField name="id" type="string">
      Audit log entry ID
    </ResponseField>
    <ResponseField name="entity_type" type="string">
      Type of entity changed (e.g., "user", "role")
    </ResponseField>
    <ResponseField name="action" type="string">
      Action performed
    </ResponseField>
    <ResponseField name="old_value" type="object" optional>
      Previous value before change
    </ResponseField>
    <ResponseField name="new_value" type="object" optional>
      New value after change
    </ResponseField>
    <ResponseField name="changed_by" type="string" optional>
      User ID who made the change
    </ResponseField>
    <ResponseField name="changed_at" type="string">
      ISO 8601 timestamp of change
    </ResponseField>
  </expandable>
</ResponseField>
